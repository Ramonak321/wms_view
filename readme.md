#Данное приложение отображает статусы заказов на табло#

##При создании расходного ордера регистрирует связь ордера с заказом, присваивает статус ордера = Запланирован##

1. Методом POST передается Реализация ордер и Номер заказа к нему (связь 1 к 1)
  1. Поля {"заказ":"№ заказа", "ордер":"№ ордера"}  
2. Кортес методом POST может отменить ордер
3. Кортес может методом POST создать объединенный ордер со связью список ордеров
4. После регистрации присваивает статус = Запланирован 


##При начале выполнения сборки статус ордера = Собирается##


По окончанию сборки статус ордера = Отобран

При полчучении реализации в бухгалтерии статус ордера = Оформление накладных

При подтверждении получения заказа статус ордера = Отгружен

При отмене ордера = статус отменен

Методом get получать все ордера
